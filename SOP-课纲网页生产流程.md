## 二、标准工作流程（每门课执行一次）

### Step 1：提供原始课纲（你负责）

**输入：** Word 版课程大纲（.docx），直接发给我即可

**不需要你自己提取内容**——把原始文件给我，我来梳理。

---

### Step 2：课纲梳理与三级架构设计（我主导，你确认）

**这是最关键的一步。** 不是所有课纲都有清晰的三层结构，需要共同判断。

**我的工作：**

1. **通读原始课纲**，理解课程整体逻辑
2. **提取并构建三级架构：**
   - **L1（课程层）**：课程名称、总时长、简介、目的、收益、适合人群
   - **L2（模块层）**：将课程内容归纳为 4-6 个模块，每个模块提炼核心目标、3条亮点、是否可单独开设0.5天
   - **L3（知识点层）**：每个模块下拆解 3-4 个具体知识点，每个知识点用一句话概括
3. **价值判断建议**：标注哪些模块**适合独立成半天工作坊**（客户愿意买单的）

**你的确认：**

1. 检查模块划分是否符合**实际授课逻辑**
2. 判断哪些模块**值得单独推广**（标为0.5天可独立开设）
3. 调整知识点描述，确保**客户看得懂、愿意买单**
4. 补充或删减内容

**输出：** 确认后的三级架构文本，双方对齐后进入下一步

---

### Step 3：讲师信息提取（协作）

在课纲架构确认后，补充讲师信息：

| 信息项 | 说明 |
|--------|------|
| 姓名 | 中文名 |
| 英文名 | 如有 |
| 身份标签 | 2-4个，每行一个，每个不超过8个字 |
| 专业背景 | 1段 |
| 核心优势 | 1段 |
| 服务领域 | 1段 |
| 照片 | 从docx中提取或另行提供，建议400x400以上正方形 |

---

### Step 4：网页制作（我负责）

**我的工作：**

1. 基于现有模板 `index.html` 修改数据
2. 调整节点（L1课程、L2模块、L3知识点）
3. 制作讲师卡片（照片 + 标签 + 介绍）
4. 本地预览调试

**交付物：** 本地 `index.html` 文件，你打开预览

---

### Step 5：确认与微调（协作）

**常见需要调整的点：**
- 讲师卡片布局（标签/介绍的排版）
- 节点颜色
- 文字内容修正
- 间距/字体大小微调

**方式：** 你打开本地 HTML 预览，提出修改意见，我即时调整

---

### Step 6：部署上线（我负责）

**前提：** 你的电脑已开启 VPN，能访问 GitHub

**操作步骤：**

1. 在 GitHub 上创建新仓库
   - 仓库名建议：`课程英文名-kg` 或 `课程简称-knowledge-graph`
   - 选 Public，不勾选 README

2. 我推送代码到 GitHub

3. 开启 GitHub Pages
   - Settings → Pages → Source 选 master/(root) → Save

4. 等待 1-2 分钟，获得链接：
   ```
   https://July-zhong.github.io/仓库名
   ```

**交付物：** 可分享的网页链接

## 四、批量生产效率优化建议

### 1. 复用模板（最高效）
- 当前 `F:\bbovis-feedback-course\index.html` 就是最佳模板
- 后续只需修改数据部分（nodes 数组和讲师信息），CSS/布局完全不变
- 同系列课程（同一位讲师）只需替换内容，讲师卡片复用

### 2. 标准化内容格式
- 每门课按统一格式提供内容，减少来回确认
- 建议模板见下文「内容提交模板」

### 3. 批量处理照片
- 讲师照片统一命名为 `instructor.jpg`
- 尺寸建议 400x400 像素以上，正方形，面部清晰

### 4. 命名规范
- 仓库名：全英文小写，用连字符，如 `feedback-course-kg`
- 链接更友好，也便于管理

## 六、常见问题 Q&A

**Q1：GitHub 链接在国内访问慢怎么办？**
A：GitHub Pages 偶尔慢，但你的项目很小（37KB），加载很快。如果客户反馈慢，后续可迁移到腾讯云 COS。

**Q2：同一门课有多个版本（1天/2天）怎么办？**
A：创建两个仓库，如 `feedback-course-1day` 和 `feedback-course-2day`，分别部署。

**Q3：课程更新后如何修改？**
A：把修改内容发给我，我更新代码 → 推送 GitHub → 等1-2分钟刷新链接即可，无需重新开启 Pages。

**Q4：没有讲师照片怎么办？**
A：可以用纯色占位图，或从讲师 LinkedIn/社交媒体获取。实在不行可暂用文字头像。
